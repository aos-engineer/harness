# Final Positions — One Statement Each
## Meridian Analytics Council | API Platform vs. SaaS Depth
### Session: 2026-03-24 | Round: Closing

---

## Catalyst

Ship a read-only Export API to the 5 named accounts as a private beta — not the full platform — 
because $280K ARR with an explicit deadline is the one data point in this decision that is not 
an assumption, and the private beta is the cheapest possible test of whether enterprise demand 
is real enough to justify everything that comes after it.

---

## Navigator

Pursue Option B as the primary engineering investment with a formally scoped Export API rider 
— because the AI-assisted insights layer is the only open, defensible, and distribution-ready 
positioning move available to us today, and a read-only export API satisfies the retention 
pressure without consuming the engineering cycle we need to claim that lane before a funded 
competitor does.

---

## Strategist

Endorse the Export API as a private beta under four non-negotiable conditions — requirements 
validation before engineering starts, named PM allocation for technical pre-sales, a commercial 
kill condition at Stage 2 completion, and board pre-commitment to enterprise AE headcount if 
the beta validates — because staged commitment with explicit gates is the only way to make a 
sound strategic bet when both the demand signal and the distribution path remain unverified.

---

## Operator

Approve the private beta on the condition that Stage 3 general availability is treated as a 
separate decision gated on a named CS owner for API support, a documented escalation path, API 
observability in production, and an enterprise AE in-process — because the hidden cost that 
kills API launches is not the build, it is the first enterprise production incident that hits 
a support team that was never sized or trained to handle it.

---

## Auditor

The council has correctly converged on a scoped Export API as superior to both Option A and 
Option B in isolation — but the single unverified assumption that still carries the most 
structural risk is not the GTM gap, not the technical estimates, and not the support burden: 
it is that neither at-risk customer has confirmed they would accept a read-only export API 
rather than full programmatic control, and if that confirmation is not obtained before Q3 
engineering begins, the entire private beta rationale collapses on the first customer call.
